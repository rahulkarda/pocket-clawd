# Bug Backlog — pocket-clawd 0.4.0

This document is the **post-release audit backlog** for `pocket-clawd@0.4.0`. It was generated by a parallel multi-angle audit (security / correctness / UX / tools / concurrency), run on commit `55f8709`, then adversarially verified — every finding listed here was confirmed by a second agent against the actual source.

## Status

| Total confirmed | Fixed in 0.4.0+ | Tracked here (open) |
|---|---:|---:|
| 36 | 11 | 25 |

The 11 already-fixed items landed in commits between `55f8709` (initial 0.4.0 README) and the post-audit fix wave — see `git log` for individual fix commits. The 25 here are intentionally deferred to a follow-up pass.

## Triage

I'm tracking these in plain markdown rather than GitHub Issues because (a) it's a single-maintainer repo, (b) git log lets you walk the fix history, and (c) the file is co-located with the code under audit. If we open the project up, these become Issues.

Each entry has: severity, file, repro, and the proposed fix verbatim from the audit's adversarial verifier.

## High severity — 4 open

### H-1. `Show on all spaces` toggle requires app restart, UI doesn't say so

- **File:** `src/renderer/apps/Settings.tsx`
- **Repro:** Toggle "Show on all spaces" off → close Settings → avatar still visible on all spaces.
- **Why:** The setting is read at avatar-window creation time only. There's no IPC to call `win.setVisibleOnAllWorkspaces()` after the toggle changes.
- **Fix:** Either (a) add an IPC channel that re-applies the setting live via `getAvatarWindow().setVisibleOnAllWorkspaces(v, {visibleOnFullScreen: true})`, or (b) add a "Restart required" hint under the toggle and gate the change behind a confirmation. (a) is the right fix.

### H-2. Hotkey input has no validation; bad accelerators leave user with no working hotkey

- **File:** `src/renderer/apps/Settings.tsx`
- **Repro:** Type `garbage` into the hotkey field → blur → `globalShortcut.register` returns `false`, app silently has no hotkey, user has no way to discover this.
- **Fix:** When `applyHotkeyFromSettings` returns `false` (already a return value from `registerHotkey`), surface it back to the renderer over IPC and show a red "Hotkey rejected — using previous" inline. Better: a real keycombo recorder component instead of free text.

### H-3. Toggling memory or web_search mid-conversation produces a cryptic API error

- **File:** `src/main/anthropicClient.ts`
- **Repro:** Mid-conversation, open Settings, toggle "Persistent memory" off. The next message has a different `tools` array than prior turns. The API rejects because tool definitions changed without a fresh conversation, in the form of a cryptic error pinned to the current turn.
- **Fix:** Lock the tools array for the current chat session. Either (a) compute tools once at chat-window-open time and reuse for every turn, or (b) detect the toggle in IPC and force-close the chat window (which kills the conversation cleanly).

### H-4. In-flight Anthropic stream never aborted on app quit; spec-write can race shutdown

- **File:** `src/main/ipcHandlers.ts`
- **Repro:** During a long streaming response, hit Cmd+Q. App exits with the HTTPS socket to api.anthropic.com still open AND `writeSpec` potentially mid-write of the .spec.md file → truncated transcript on disk.
- **Fix:** (1) Hold an `AbortController` per in-flight chat in ipcHandlers.ts, pass `controller.signal` to `streamChat`, abort it in `app.on('before-quit')`. (2) Track in-flight `writeSpec` Promise and `await` during shutdown (with a short timeout) before letting the process exit — call `e.preventDefault()` once in before-quit, await pending work, then `app.exit(0)`.

## Medium severity — 13 open

### M-1. 10 MB total-size cap only enforced in `cmd_create`, not `str_replace` / `insert`

- **File:** `src/main/memory.ts`
- **Repro:** Create 200 files at 50 KB each (10 MB total — borderline-allowed), then `str_replace` each one to 100 KB. Each individual write passes the per-file cap; total store grows to 20 MB undetected.
- **Fix:** In `cmd_str_replace` and `cmd_insert`, after computing the new content, also compute `deltaBytes = byteLength(next) - byteLength(body)` and reject if `(await getTotalSize()) + deltaBytes > MAX_TOTAL_BYTES`.

### M-2. `SETTINGS_UPDATE` accepts arbitrary `outputDir` from renderer with no validation

- **File:** `src/main/ipcHandlers.ts`
- **Repro:** A compromised renderer (or future renderer feature) sends `outputDir: '/etc'` via IPC. The clean handler clamps strings to 500 chars but doesn't validate that `outputDir` is a writable directory under user home.
- **Fix:** In the `clean.outputDir` branch, reject if path doesn't start with `app.getPath('home')` or doesn't exist as a directory. Use the existing dialog flow as the only canonical setter.

### M-3. Avatar size slider commits only `onMouseUp`

- **File:** `src/renderer/apps/Settings.tsx`
- **Repro:** Use keyboard arrows on the slider, or touchpad pointer-leave during drag → setting visually updates but doesn't persist.
- **Fix:** Switch to `onChange` with debounce (200ms) instead of `onMouseUp`.

### M-4. Whisper Min can exceed Max in Settings

- **File:** `src/renderer/apps/Settings.tsx`
- **Repro:** Type `30` in min, `5` in max → no UI clamp. (The whisperEngine NaN-guard catches some of this but doesn't prevent the inversion itself.)
- **Fix:** Cross-validate on each change: `whisperIntervalMin = Math.min(min, max)`, `whisperIntervalMax = Math.max(min, max)`.

### M-5. `onBlur`-only saves are lost when Settings window closes without unfocusing

- **File:** `src/renderer/apps/Settings.tsx`
- **Repro:** Type a new persona text → click the X to close Settings → text not saved (onBlur didn't fire because focus went to a destroyed window).
- **Fix:** Add a `'before-close'` window listener in main that sends an IPC event the renderer responds to with a forced flush; or save on every keystroke (debounced 500ms).

### M-6. First-run: after adding API key, whispers don't start for up to 30s

- **File:** `src/main/index.ts`
- **Repro:** Fresh install → app opens Settings → user pastes key → for the next 30 seconds the whisper engine is stopped because it's polling on a 30s interval.
- **Fix:** When `setApiKey` IPC handler succeeds, immediately call `startWhisperIfNeeded()` from the IPC layer instead of waiting for the watchForApiKey poll.

### M-7. First-run experience: Settings opens cold with no welcome/orientation

- **File:** `src/main/index.ts`, `src/renderer/apps/Settings.tsx`
- **Repro:** Fresh install → Settings opens with a blank API key field, no greeting, no explanation.
- **Fix:** Detect `!settings.onboarded` in Settings.tsx and show a top-of-window welcome card explaining: "Paste your Anthropic key (stored in Keychain), then close this window — Clawd will appear in your menubar." Set `onboarded: true` after the key is saved.

### M-8. Custom `baseURL` has no validation or test

- **File:** `src/main/anthropicClient.ts`
- **Repro:** User pastes a malformed URL (`"htp://example.com"`) → no immediate feedback, only on next chat.
- **Fix:** Add a "Test" button next to the baseURL field that fires a `client.messages.count_tokens()` request and reports success/failure inline.

### M-9. Picking a new output dir abandons old transcripts to `search_past_sessions`

- **File:** `src/main/ipcHandlers.ts`
- **Repro:** User picks a new output dir → `search_past_sessions` no longer finds old transcripts.
- **Fix:** When changing outputDir, prompt the user: "Move existing N session files to the new location? [Move] [Leave]". On Move, copy + delete the originals.

### M-10. `web_search_20260209` was the dynamic-filtering variant — model-compat caveat

- **File:** `src/main/anthropicClient.ts`
- **Status:** Mooted by the switch to a client-side `web_search` custom tool (commits in 0.4.1). The hosted-tool registration was removed.

### M-11. Memory tool return-value strings diverge from the spec format

- **File:** `src/main/memory.ts`
- **Repro:** Anthropic's `memory_20250818` spec returns specific strings like `"File created successfully at <path>"` for `create`, etc. Our handlers return `"Created <path> (<bytes> bytes)"` — works but isn't the spec format the model was trained against.
- **Fix:** Align return strings with Anthropic's published memory tool reference output format. Likely doesn't affect functionality but improves model behavior consistency.

### M-12. Whisper-restore `setTimeout`: stale `prev` closure + untracked timer

- **File:** `src/main/index.ts`
- **Repro:** Two whispers within 1200ms (idle-alert collision with regular tick) → first timer's `prev='idle'` fires at t=1200, restoring state to idle while whisper #2 is mid-animation, truncating it.
- **Fix:** Module-scoped `whisperRestoreTimer` cleared on each new whisper and in `will-quit`. Replace the `prev`-based ternary with a single `animLockUntil` timestamp owned by the latest whisper.

### M-13. Drag offset captured during animated edge-snap returns mid-animation position

- **File:** `src/main/avatarWindow.ts`
- **Repro:** Drag avatar near edge → release (snap animation begins) → grab again within 150ms → window jumps because `getPosition()` returned the in-flight intermediate.
- **Fix:** Pass `animate=false` to `setPosition` in `endDrag` (the snap distance is small, animation is purely cosmetic). One-character fix.

## Low severity — 8 open

### L-1. Several IPC handlers trust renderer types without runtime checks

- **File:** `src/main/ipcHandlers.ts`
- **Repro:** Hostile renderer sends `[]` where a string is expected, etc. Settings sanitizer covers most but tool inputs and a few others don't.
- **Fix:** Add `zod` or hand-rolled type guards at every IPC handler entry point.

### L-2. Unawaited async `onDone` work — partially fixed in 0.4.1

- **File:** `src/main/ipcHandlers.ts`
- **Status:** The high-severity version was fixed by changing `StreamCallbacks.onDone` to `Promise<void>` and awaiting at all call sites. This residual is just the unused `{ok, full}` return-shape inaccuracy. Cosmetic.

### L-3. "Open at login" toggle stays in user-wanted state even when macOS rejected

- **File:** `src/renderer/apps/Settings.tsx`
- **Status:** The honest banner does surface this; the toggle's visual state could additionally indicate "wanted but not actual" with a different color. Polish.

### L-4. "Open memory folder" button gives no feedback if it fails

- **File:** `src/main/ipcHandlers.ts`
- **Repro:** Memory dir doesn't exist or `shell.openPath` returns an error → silent.
- **Fix:** Catch and return error message; renderer surfaces it via flash or alert.

### L-5. "Saved" flash fires for settings that don't take effect

- **File:** `src/renderer/apps/Settings.tsx`
- **Repro:** Toggle showOnAllSpaces → "Saved" flash, but setting doesn't apply (see H-1).
- **Fix:** Make flash conditional on `actually applied` — main returns a `{applied, requiresRestart}` flag.

### L-6. `insert_line` out-of-range silently clamps

- **File:** `src/main/memory.ts`
- **Repro:** Claude requests `insert_line: 9999` on a 5-line file → clamps to 5 silently.
- **Fix:** Refuse with explicit error so Claude can read the file again with view_range and use a real line number.

### L-7. Total-size check is racy with concurrent writes

- **File:** `src/main/memory.ts`
- **Repro:** Two parallel `cmd_create` calls each compute `getTotalSize()` before either has written. Both pass the cap check, both write, total exceeds cap.
- **Fix:** Wrap memory ops in a per-store async mutex (a chained Promise). Alternatively, accept the race — the cap is soft.

### L-8. `flash()` and `refreshLoginItem` setTimeout calls fire setState after unmount

- **File:** `src/renderer/apps/Settings.tsx`
- **Status:** Mostly shadowed by the BrowserWindow being destroyed on close, so unmount-after-setState rarely fires in production. Latent — would matter under HMR or future refactor that mounts/unmounts within the same renderer.
- **Fix:** Track timer handles in refs, clear in `useEffect` cleanup.
